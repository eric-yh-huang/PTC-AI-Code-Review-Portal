📜 審查規範與規則

所有 AI 審查皆基於 Back-end 團隊統一維護的全域規則。

📂 審查基準 (Baseline) 儲存庫架構

所有的審查規則與指令 (Prompt Rules) 統一由 Admin Tool 儲存庫進行版本管理。這確保了跨團隊審查的一致性：

AdminTool/Tools/ReviewAgents/
├── rules/
│   ├── backend/    (後端規則: General, Coding Standards...)
│   ├── frontend/   (前端規則)
│   └── infra/      (基礎設施規則)
├── shared/         (共享資源: 格式模板, 詞彙表)
└── pipelines/      (範例流水線與腳本)
🔹 Rules/Backend: 存放了從 00 到 99 編號的 Markdown 規範，AI 會依序加載這些邏輯。
🔹 Shared: 定義了 ReviewFormat.Template.md，這決定了您在 PR 留言中看到的 UI 排版。
🔹 Version Control: 所有的規則變動都需經過 PR，實現「Rules as Code」。

當您在 Pipeline 中配置 inputGlobalRulesPath 時，實質上就是將 AI 的「思維邊界」指向了這個結構化的目錄。

🛠️ 如何自訂規則

若個別專案有特殊需求，可以在 Pipeline 設定中透過 `inputSystemInstruction` 加入額外的指令，但建議優先使用全域規則以維持一致性。
