理解 PTC AI 審查體系的決策路徑與資源調度細節。

🏗️ Module-ORD 專案架構

本專案採用典型的 .NET 分層架構，實現了關注點分離 (Separation of Concerns)：

Module-ORD/
├── GeneralOrderController (API 層)
├── GeneralOrderService   (業務邏輯層)
├── Repository           (數據訪問層)
├── Common               (共享組件層)
├── GeneralOrderUnitTest (單元測試)
└── cicd                 (自動化腳本)
🔹 Controller: 負責路由分發與 API Entry Points。
🔹 Service: 核心邏輯實現，採用 Interface 進行解耦。
🔹 Repository: 處理 Persistence 與資料持久化。
🔹 Common: 存放 DTO、Infrastructure 與共用 Helper。
🔹 UnitTest: 確保代碼邏輯的正確性與穩定性。
