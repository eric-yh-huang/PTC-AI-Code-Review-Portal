🚀 接入指南

只需四個步驟，即可為您的專案綁定自動化 AI 代碼審查服務。

Step 1: 建立 YAML 與配置基準 (Baseline) 儲存庫

在您的儲存庫 `cicd/` 目錄下建立一個新的 YAML 檔案（例如 `ai-code-review.yml`）。

💡 重點說明： 為了確保所有專案的審查標準一致，Pipeline 必須宣告 AdminTool 作為外部資源 (Resources)。這樣在執行時才能跨儲存庫讀取統一的 Code Review Baseline (審查基準與提示詞)。

# 基礎設定與資源宣告
resources:
  repositories:
    - repository: AdminTool              # 定義識別名稱
      type: git
      name: Back-end/Admin Tool          # 目標儲存庫 (存放審查規則)
      ref: dev/1.54

pr:
  - dev/1.54 # 設定觸發 PR 的分支

pool:
  name: "Back-end_Self-Hosted_K8s"

Step 2: 載入規則與 AI 審查 Job

在 Pipeline 步驟中，除了主程式碼，還必須 checkout: AdminTool 才能讀取規則，接著再呼叫 `PtcWebScAiCodeAgent@1` 任務：

steps:
  - checkout: self       # 載入當前要被審查的 PR 程式碼
  - checkout: AdminTool  # 載入全域審查規則
    path: AdminTool

  - task: PtcWebScAiCodeAgent@1
    inputs:
      inputAiProvider: "Ollama"
      inputOllamaModelName: "qwen3-coder:latest"
      # 指定剛剛 checkout 下來的規則路徑
      inputGlobalRulesPath: '$(Agent.BuildDirectory)/AdminTool/Tools/ReviewAgents/rules/backend'

Step 3: 配置安全性權限與資源

在執行 Pipeline 前，必須先在 Azure DevOps 的後台完成以下兩項配置：

🛡️ 1. 上傳 GCP 認證密鑰 (Secure File)

路徑：Pipelines > Library > Secure files 頁籤

  • 點擊 + Secure file 上傳 websc-dev-433809-5accf54ec817.json。
  • 上傳後點擊檔案進入設定，確保 Pipeline permissions 已授權給您的 Pipeline。

🔑 2. 設定偵測權杖 (Pipeline Variables)

路徑：進入 Pipeline 編輯畫面 > 點擊右上角 Variables 按鈕

  • 點擊 + (New variable)。
  • Name 輸入 OllamaPoolPAT。
  • 勾選 Keep this value secret (確保權杖不外洩)。
  • ⚠️ PAT 權限範圍僅需設定為：Agent Pools (Read)。

Step 4: Azure DevOps PR 策略綁定 (Build Validation)

YAML 寫好並推送到遠端後，必須在 Azure DevOps 後台設定強制觸發，請依序操作：

1 前往 Project Settings (專案設定)。
2 在左側選單點擊 Repos > Repositories。
3 選擇您的目標 Repository，然後切換到 Policies 頁籤。
4 捲動到下方點擊目標保護分支 (例如 main 或 dev/1.54)。
5 找到 Build Validation 區塊，點擊 + (Add)。
6 在 Build pipeline 下拉選單中，選擇剛剛建立的 AI Code Review Pipeline 並儲存。

🎉 完成！以後任何人對該分支發起 Pull Request，都會自動觸發 AI 審查了！
